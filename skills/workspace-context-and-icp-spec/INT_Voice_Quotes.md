# INT — Voice Quotes
*Stage 2. Preserve the client's actual language. Primary source for ToV.*

Pull verbatim lines. Goal: Stage 3 writes in the client's voice instead of inventing one.

Rules:
- Quote exactly. Don't fix grammar or tighten phrasing.
- Every quote cites a URL or filename.
- Tag with one or more: `positioning` / `proof` / `pain` / `objection` / `vocabulary` / `anti-claim-seed`.

---

## Source priority by tier

**Tier 0 (baseline):** website hero / About / Product / Pricing copy + company LinkedIn page About + customer logos pages.
This yields the *formal* register — usually clean and polished, often missing the client's actual voice.

**Tier 1 (when available):** founder LinkedIn posts (up to 20, none older than 12 months), founder podcasts, conference talks, comment threads under founder posts.
This is where the *real voice* shows up — looser, sharper, opinionated. Tier 1 is what makes ToV in Project Context §3 evidence-backed rather than inferred.

**Tier 2 (when client provides):** interview transcripts, internal decks, recorded sales calls.
Highest-yield for pain quotes and anti-claim seeds.

On Tier 0 only: this file is short. That's correct — don't pad.
